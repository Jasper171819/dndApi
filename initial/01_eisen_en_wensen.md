# Eerste lijst van eisen en wensen

## Doel

De applicatie zal een kleine website worden waarmee een gebruiker D&D karakters zal kunnen beheren.

## Eisen

1. De gebruiker zal een lijst met karakters kunnen zien.
2. De gebruiker zal 1 karakter kunnen bekijken.
3. De gebruiker zal een karakter kunnen toevoegen.
4. De gebruiker zal een karakter kunnen aanpassen.
5. De gebruiker zal een karakter kunnen verwijderen.
6. De frontend zal de eigen Laravel API gebruiken.
7. De API zal JSON teruggeven.
8. De database zal de karaktergegevens kunnen opslaan.

## Wensen

1. De lijst met karakters en het formulier zullen op dezelfde pagina staan.
2. Bij elk invoerveld zal een duidelijk label staan.
3. De gebruiker zal een melding krijgen na opslaan, aanpassen of verwijderen.
4. Er zal voorbeelddata zijn om de app snel te testen.
