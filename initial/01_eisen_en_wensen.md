# Eerste lijst van eisen en wensen

## Doel

De applicatie zal een kleine website worden waarmee een gebruiker D&D characters zal kunnen beheren.

## Eisen

1. De gebruiker zal een lijst met characters kunnen zien.
2. De gebruiker zal 1 character kunnen bekijken.
3. De gebruiker zal een character kunnen toevoegen.
4. De gebruiker zal een character kunnen aanpassen.
5. De gebruiker zal een character kunnen verwijderen.
6. De frontend zal de eigen Laravel API gebruiken.
7. De API zal JSON teruggeven.
8. De database zal de charactergegevens kunnen opslaan.

## Wensen

1. De applicatie zal op 1 pagina kunnen werken.
2. De pagina zal een duidelijk formulier hebben.
3. De gebruiker zal een melding krijgen na opslaan of verwijderen.
4. Er zal voorbeelddata zijn om de app snel te testen.
